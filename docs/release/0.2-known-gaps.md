# 0.2 Known Gaps and Follow-up Backlog

This document is the honest companion to `CHANGELOG.md` for the 0.2
release. It enumerates issues surfaced by the 0.2 ship-readiness review
that did not block the release but are being tracked for 0.2.x patches
or 0.3.

The intent is that no review-flagged issue silently disappears between
releases. Each entry names the surfacing concern, where it lives, and
the planned resolution window.

## Calibration corpus follow-ups

- Match key `(Type, File)` ignores `Symbol`; multi-symbol fixtures can't distinguish "detector fired per-symbol" from "detector fired once with both signals on the same line." Add Symbol to the key.
- `ExpectedAbsent` matching path uses non-normalised `sig.Location.File`; broken for eval-data detectors that stamp absolute paths. Use the same normalisation as positive matching.
- No fixture exercises the suppression branch of any "missing X" detector (e.g. `eval-with-temperature-pin/`, `skipped-with-ticket/`). Add paired negatives.
- No DeepEval or Ragas fixtures; only Promptfoo. Add at least one of each.
- All threshold-bracketing fixtures (cost, retrieval, coverage) overshoot by orders of magnitude; no near-threshold cases. A comparator-flip regression would survive.
- Runtime fixtures (`flakyTest`, `slowTest`, etc.) label `file:` as JUnit classname rather than file path; brittle to JUnit parsing improvements.
- `t.Skipf` if the corpus directory is empty; the load-bearing gate is bypassable by deletion. Replace with `t.Fatalf` and a minimum-fixture-count assertion.
- `aiHardcodedAPIKey` has no fixture (constructing one requires a high-entropy non-example string that GitHub Secret Scanning would alert on).

## CLI restructure follow-ups (0.2.x or 0.3)

- **In-band deprecation warnings** on legacy top-level commands. Without these, users won't migrate to the canonical shape before 0.3 removes the legacy names.
- **`terrain --help` rewrite** to lead with the canonical 11; today still prints the 35-command surface.
- **`terrain migrate --help` / empty-args** prints the legacy `migrate <dir>` usage block, not the namespace-verb listing.
- **`terrain migrate config --help`** prints `Usage: terrain convert-config ...` (legacy command name).
- **`did you mean` suggestions** prefer legacy commands over canonical (e.g. `terrain insihgts` suggests `insights`, not `report insights`).
- **End-to-end equivalence tests** between legacy and namespace paths. Today the namespace tests verify dispatcher routing only.
- **Phase B (deferred to 0.3)**: fold `policy` into `analyze --policy=<file>`, `compare` into `analyze --against=<ref>`, ship `--focus=<path>` and `--output=<path>` flags with runner-side support.

## Distribution / supply-chain follow-ups

- **`bin/terrain-installer.js`** silently degrades to checksum-only when `cosign` is absent. Promote to mandatory with `TERRAIN_INSTALLER_SKIP_VERIFY=1` documented escape; 0.2.x.
- **`docs/release/supply-chain.md`** still uses `v0.1.0` examples; needs `v0.2.0` refresh and a `gh attestation verify` section for SLSA L2.
- **Homebrew formula updater** is not gated on release-workflow success; a partial release could publish a brew formula pointing at missing platform archives.

## Documentation follow-ups

- **`docs/release/feature-status.md`** still says "Feature Status — 0.1.2"; not updated for 0.2. The 0.1.2 release explicitly promised this doc would become a release-blocker gate; that promise is unkept until it's rewritten and brought under `docs-verify`.
- **`README.md`** says "30+ commands" with 0.1.2 framing; needs refresh for the canonical 11-command shape.
- **`docs/schema/COMPAT.md`** says current snapshot schema is 1.0.0; actual is 1.1.0.
- **`docs/scoring-rubric.md`** references `--scoring=v3`; the round-4 plan + CHANGELOG call it `v2`. Pick one.
- **`docs-verify` scope** — only checks manifest, severity rubric, and rule doc stubs. Does not check `feature-status.md`, `README.md`, `COMPAT.md`. Expand or be honest that the gate is manifest-only.

## How this list was produced

`/gambit:parallel-agents` adversarial review on the 0.2 ship-readiness
PRs, dispatched across five independent domains (AI detectors,
calibration corpus, CLI restructure, eval adapters, documentation).
~150 findings; the ship-blockers were fixed in the corresponding
release-prep commit; everything else is captured here.
