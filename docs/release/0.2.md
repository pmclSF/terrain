# Release 0.2 — Taxonomy + AI Moat Foundation

**Status:** shipping (release 0.2.0 — see CHANGELOG.md and `0.2-known-gaps.md` for what landed in-band vs deferred to 0.3)
**Theme:** Lay the foundation that makes Terrain credibly AI-native and unlocks calibrated scoring. Schema, taxonomy, calibration corpus, tree-sitter pooling, and the first wave of new AI signals.
**Target window:** 6 weeks of focused work (~80 SP).

0.1.2 truthed up the product description and locked schemas. 0.2 is where Terrain starts shipping the **moat features** that distinguish it from Promptfoo, DeepEval, Codecov, Sonar, and Launchable: explainable AI signals, calibrated severity, and structural-graph impact selection that no other tool offers.

---

## Goals

1. AI-native readiness lifts from ~38% (post 0.1.2) to **~65%**.
2. SignalV2 schema with multi-axis taxonomy ships; back-compat shim keeps v1 readable.
3. Severity rubric document (single source of truth) published. Every detector cites the clauses justifying its severity.
4. Calibration corpus v1 live (per-detector fixtures, 27 in 0.2.0); ≥95% recall on labeled fixtures is a load-bearing release gate. The 50-repo labeled-corpus + ≥90% precision floor slipped to 0.3 — see `0.2-known-gaps.md` "Calibration corpus follow-ups."
5. Tree-sitter parser pool eliminates per-file parser allocation; hierarchical Go `t.Run` and JUnit 5 `@Nested` work correctly.
6. Top-3 conversion directions (Cypress→Playwright, Jest→Vitest, Mocha→Jest) reach A-grade with 95% post-conversion test pass rate.
7. CLI command surface restructured: 43 → 11 canonical commands (originally targeted ~15; landed at 11) via the `report`/`migrate`/`convert`/`config`/`debug` namespaces; legacy aliases retained through 0.2.x with opt-in deprecation hints (`TERRAIN_LEGACY_HINT=1`).
8. Cosign signature verification on the npm postinstall path is now mandatory (warn-only in 0.1.2 → hard-fail in 0.2).
9. SLSA provenance level 2 attestations published with every release.
10. Multi-OS CI matrix gates merges; performance regression suite establishes baseline.

## Critical path

The 0.2 work is mostly parallelizable, but five items must land in this order:

1. **Schema lock + manifest.go gates expanded** (carries from 0.1.2; CI gate becomes hard-fail in 0.2).
2. **SignalV2 schema designed and migrated**. All detectors emit v2; v1 read-shim available.
3. **Severity rubric published and enforced**. Detector commits must cite rubric clauses.
4. **Calibration corpus v1 live**. ~50 repos with ground-truth labels; corpus gate added to CI.
5. **Tree-sitter parser pool**. Unblocks discovery, conversion, and AI surface inference improvements.

Items 2–5 can run in parallel after 1. Items 6+ in §"Work items" can run in parallel after 5.

## Work items by area

### AI Workflows — priority

**Surface detection expansion**

- YAML/JSON prompt detection: parse promptfoo configs, RAG configs, embedded `tests[].prompt`. Adds prompt surfaces from non-Python/JS sources.
- Dataset filename + DB cursor detection: `.jsonl`, `.parquet`, `.csv`, `.arrow`, `TFRecord`, plus `psycopg2.fetch*`, `pymongo.find()`, `client.search()`.
- RAG surfaces for non-framework implementations: pgvector, Elasticsearch vector search, Weaviate REST, in-memory FAISS.
- Tool/schema dynamic builders: detect runtime-assembled JSON schemas; recognize MCP tool definitions.

**12 new AI signal detectors** (all new — none exist today)

| Signal | Severity | Detection |
|---|---|---|
| `aiSafetyEvalMissing` | High | Agent/prompt with no eval scenario covering safety |
| `aiPromptVersioning` | Medium | Prompt content changed without version bump in metadata or filename |
| `aiPromptInjectionRisk` | High | User input concatenated into prompt without escaping |
| `aiHardcodedAPIKey` | Critical | API key pattern in eval YAML / config / source |
| `aiToolWithoutSandbox` | High | Destructive tool (delete/exec) lacks approval gate or sandbox |
| `aiNonDeterministicEval` | Medium | LLM call without `temperature=0` or seed pinning |
| `aiModelDeprecationRisk` | Medium | Model pinned to deprecated tag (e.g., `gpt-3.5-turbo`) or floating tag |
| `aiCostRegression` | Medium | Prompt change increases token count >25% vs baseline |
| `aiHallucinationRate` | High | Eval reports fabricated outputs above threshold |
| `aiFewShotContamination` | Medium | Few-shot examples overlap with eval test set |
| `aiEmbeddingModelChange` | Medium | Embedding model swap detected without RAG re-evaluation |
| `aiRetrievalRegression` | High | Context relevance / nDCG / coverage dropped vs baseline |

**Eval framework integration**

- Promptfoo: parse run results (accuracy, latency, cost per case).
- DeepEval: parse artifact format; extract custom metric values.
- Ragas: ingest `retrieval_score`, `faithfulness`, `answer_relevancy`.
- Plugin architecture skeleton (`internal/airun/plugin.go` interface) for community adapters.

**`terrain ai run` actually executes**

- Replace placeholder with real eval execution path.
- Parse stdout/artifact; persist metrics to baseline.
- Wire `terrain ai compare` to detect regressions.

### Test Discovery

- `sync.Pool` per language for tree-sitter parsers (eliminates 1000+ allocations on 1k-file repos).
- Hierarchical Go `t.Run` extraction (multi-level `SuiteHierarchy`).
- JUnit 5 `@Nested` + `@DisplayName` extraction.
- Pytest `parametrize` value extraction (currently estimates count only).
- Pytest fixture dependency graph (conftest discovery + transitive resolution).
- TypeScript `tsconfig.json` `paths` consistent across import resolution.
- Vitest in-source tests (`if (import.meta.vitest)` blocks).

### Test Conversion

- Top-3 directions to A-grade with 95% post-conversion pass rate:
  - Cypress→Playwright (50-fixture corpus)
  - Jest→Vitest (40-fixture corpus)
  - Mocha→Jest (35-fixture corpus)
- `terrain convert --preview` shows diff before applying.
- Per-file confidence reporting (`itemsCovered`, `itemsLossy`, confidence %).
- `.terrain/conversion-history/` for audit trail.

### Analysis Engine

- **SignalV2 schema** with multi-axis taxonomy:
  - Domain, severity, severity_clauses[], confidence{value, interval, quality, sources}, actionability, lifecycle_stages[], ai_relevance
  - rule_id (`TER-<DOMAIN>-<NUM>`), rule_uri, detector_version
  - related_signals[] for compound-evidence aggregation
- v1→v2 migration shim; old code reads v2 via projection.
- **Severity rubric** (`docs/severity-rubric.md`) — Critical/High/Medium/Low/Info with crisp clauses + examples.
- **Drift linters**: every `SignalXxx` constant ⇒ in `manifest.go` ⇒ has detector ⇒ documented at rule_uri. Doc-gen CI gate becomes hard-fail.
- **Calibration corpus v1**: 50 repos with manual ground-truth labels; per-detector precision/recall measured in CI.
- **Scoring v2**: documented formula replacing `max(density, log-weighted)` magic. Bands re-anchored to corpus percentiles (not hardcoded 4/9/16). Hysteresis simplified or removed.
- **Confidence intervals**: replace point estimates with Wilson/Beta intervals. Update `terrain explain` to show range, not single number.

### CLI / UX

- Section-structured human output: Headline → Findings → Profile → Next Actions.
- Glossary footer for human output (define `fanout`, `blast radius`, `weak assertion`, etc. on first use).

## Tier 2 release gates (mandatory for 0.2)

- All 0.1.2 gates remain.
- Calibration corpus v1: ≥95% recall on labeled fixtures (precision floor deferred to 0.3 with corpus v2).
- Performance budget: P50 < 10s on 10k-file repo.
- Backward compat: 0.1.2 snapshots deserialize without error in 0.2 binaries.
- Cross-platform: Homebrew formula installs on macOS; npm package installs on darwin/linux/windows.
- AI signal coverage: all 12 new detectors registered and emitting on the AI fixture corpus.
- Top-3 conversion directions: 95% post-conversion test pass rate on respective fixture corpora.
- Drift gate: zero diff on `make docs-verify`.
