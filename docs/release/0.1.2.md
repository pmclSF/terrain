## Goals

1. Every headline claim in `README.md` and `DESIGN.md` either holds in code or is explicitly tagged `[experimental]`.
2. `TestSuiteSnapshot` schema v1.0.0 is canonical; backwards-compat policy published.
3. `internal/signals/manifest.go` is the single source of truth for signal types; doc-generation pipeline scaffolded (full CI gate deferred to 0.2).
4. Signed binaries ship for darwin/linux Ã— amd64/arm64 + windows/amd64. npm postinstall has the cosign verification skeleton in place.
5. CRITICAL correctness defects from rounds 1â€“3 fixed (zero-guards, name-collision matching, prefix-overlap false positives).
6. CLI honors `NO_COLOR`, `flag.ContinueOnError` everywhere, distinct exit codes (0/1/2/3/4), did-you-mean suggestions, SIGINT cleanup.
7. Privacy/security quick wins: telemetry mode `0o600`, `--base` ref validation, SARIF redaction option.
8. CI hygiene: pinned Action SHAs, multi-OS matrix, CODEOWNERS, PR template, husky pre-commit blocks binary additions.

- SignalV2 schema migration (only v1 lock here)
- Calibration corpus (severity rubric documented; corpus deferred)
- Tree-sitter parser pooling (architectural change)
- Detector algorithmic upgrades (AST-based weakAssertion, etc.)
- Full doc-generation CI gate (scaffold only)
