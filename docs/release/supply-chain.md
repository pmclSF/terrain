# Supply-Chain Artifacts

Every tagged release of Terrain produces the following artifacts, attached to
the GitHub Release and available for download.

## Artifact Manifest

For a release tagged `v3.0.0`, the Go release job produces:

| Artifact | Format | Purpose |
|----------|--------|---------|
| `terrain_3.0.0_linux_amd64.tar.gz` | tar.gz | Linux x86-64 binary |
| `terrain_3.0.0_linux_arm64.tar.gz` | tar.gz | Linux ARM64 binary |
| `terrain_3.0.0_darwin_amd64.tar.gz` | tar.gz | macOS Intel binary |
| `terrain_3.0.0_darwin_arm64.tar.gz` | tar.gz | macOS Apple Silicon binary |
| `terrain_3.0.0_windows_amd64.zip` | zip | Windows x86-64 binary |
| `terrain_3.0.0_windows_arm64.zip` | zip | Windows ARM64 binary |
| `checksums.txt` | SHA-256 | Integrity checksums for all archives |
| `checksums.txt.sig` | Sigstore | Detached cosign signature for checksums |
| `checksums.txt.pem` | X.509 | Signing certificate (Sigstore OIDC) |
| `*.cdx.json` | CycloneDX JSON | SBOM per archive (dependency inventory) |
| `*.spdx.json` | SPDX JSON | SBOM per archive (ISO/IEC 5962:2021) |

The npm release job additionally publishes `terrain-testframework` to npm with
`--provenance` (Sigstore OIDC attestation via npm's built-in support).

## Checksums

All archive checksums are in `checksums.txt` (SHA-256, one line per file).

### Verify an archive

```bash
# Download the release archive and checksums
curl -LO https://github.com/pmclSF/terrain/releases/download/v3.0.0/terrain_3.0.0_linux_amd64.tar.gz
curl -LO https://github.com/pmclSF/terrain/releases/download/v3.0.0/checksums.txt

# Verify
sha256sum -c checksums.txt --ignore-missing
```

## SBOM (Software Bill of Materials)

Each release archive has two companion SBOM files generated by
[syft](https://github.com/anchore/syft):

- **CycloneDX JSON** (`.cdx.json`) — compatible with OWASP Dependency-Track,
  Snyk, Grype, and most commercial SCA tools
- **SPDX JSON** (`.spdx.json`) — ISO/IEC 5962:2021, compatible with SPDX
  tooling, NTIA minimum-elements compliant

### Inspect an SBOM

```bash
# View with syft
syft convert terrain_3.0.0_linux_amd64.tar.gz.cdx.json

# Scan for vulnerabilities with grype
grype sbom:terrain_3.0.0_linux_amd64.tar.gz.cdx.json
```

### Local generation

```bash
make sbom              # generates terrain.cdx.json and terrain.spdx.json
make sbom-cyclonedx    # CycloneDX only
make sbom-spdx         # SPDX only
```

## Sigstore Signing

Release checksums are signed using [cosign](https://github.com/sigstore/cosign)
keyless mode via Sigstore OIDC. The signing identity is the GitHub Actions
workflow that produced the release.

### Verify a signature

```bash
# Download checksum file, signature, and certificate
curl -LO https://github.com/pmclSF/terrain/releases/download/v3.0.0/checksums.txt
curl -LO https://github.com/pmclSF/terrain/releases/download/v3.0.0/checksums.txt.sig
curl -LO https://github.com/pmclSF/terrain/releases/download/v3.0.0/checksums.txt.pem

# Verify with cosign
cosign verify-blob checksums.txt \
  --signature checksums.txt.sig \
  --certificate checksums.txt.pem \
  --certificate-identity-regexp 'https://github.com/pmclSF/terrain/' \
  --certificate-oidc-issuer 'https://token.actions.githubusercontent.com'
```

### Signing status

Sigstore signing is enabled in CI when `COSIGN_ENABLED=1` is set in the
release workflow environment. If cosign is not available (e.g., local
`goreleaser release --snapshot`), signing is skipped and only checksums and
SBOMs are produced.

## npm Provenance

The npm package `terrain-testframework` is published with `npm publish
--provenance`, which creates a Sigstore-backed provenance attestation linking
the published package to the specific GitHub Actions workflow run, commit SHA,
and build environment.

### Verify npm provenance

```bash
npm audit signatures
# or inspect at: https://www.npmjs.com/package/terrain-testframework
```

## Dry Run

To preview all release artifacts locally without publishing:

```bash
make release-dry-run
# or directly:
goreleaser release --snapshot --clean --skip=publish,sign
```

This produces all archives, checksums, and SBOMs in `dist/` without uploading
or signing.
