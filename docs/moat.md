# Moat

Hamlet's moat is not the CLI.

The moat is the accumulated intelligence graph and the benchmark dataset built on top of snapshots and signals.

## Core compounding assets

### 1. Test intelligence graph
Relationships between:
- code units
- tests
- frameworks
- runtime behavior
- quality signals
- risk surfaces
- ownership
- policy state

### 2. Historical snapshots
Enable:
- trend analysis
- drift detection
- risk regression tracking
- quality movement tracking

### 3. Cross-repo benchmark data
Enable:
- comparison to similar organizations
- percentile context
- normal ranges for reliability, speed, and quality

### 4. Migration intelligence corpus
Enable:
- known blockers
- framework-portability patterns
- migration success prediction

### 5. Workflow entrenchment
Hamlet becomes sticky when it is:
- in the CLI
- in the IDE
- in CI
- in engineering management review loops

## What is not a moat

- codemods alone
- a basic CLI
- one-off detectors
- static rules without history or aggregation

## Strategic implication

V3 must be snapshot-first and signal-first so the data needed for the moat can accumulate immediately.
